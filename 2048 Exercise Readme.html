Overview

2048 is a single-player puzzle game in which the objective is to slide numbered tiles on a grid to combine them and create a tile with the number 2048. Your job today is to create an implemention of the game mechanics using your favorite programming language. You should not spend more than 2-4 hours on your solution. You must provide unit tests for your API. A graphical user interface is not required.

Questions regarding the exercise will not be answered; if you make any assumptions please state them when you submit your test.

Rules of 2048

  • The game board is a 4×4 grid of tiles, each tile is either occupied with a number, or empty.
  • The game board is initialized with two randomly placed tiles, each of which can be either 2 (90% probability) or 4 (10% probability).
  • You may move in one of four directions: up, left, down, right.
  • You may only move in a given direction if the move would change the state of the board.
  • While moving, the tiles will slide as far as possible in the chosen direction until they are stopped by another tile or the edge of the grid.
  • If two tiles of the same number collide while moving, they will merge into a tile with the total value of the two tiles that collided. The resulting tile cannot merge with another tile again in the same move.
  • After moving, a random empty tile is set to either 2 (90% probability) or 4 (10% probability).
  • If a 2048 tile appears anywhere on the board, the game is won.
  • If all tiles have numbers, and there is no valid move, the game is lost.

Requirements

  • Implement a minimal game board API, capable of the following:
    • Move in any valid direction
    • Win condition
    • Lose condition
    • Insert a random tile
  • Unit tests

Key Considerations

Writing good code is a balancing act between a number of (sometimes conflicting) factors. A few things to consider while writing your solution are:
  • Readability
  • Maintainability
  • Efficiency

Bonus

  • Make a playable version of the game
